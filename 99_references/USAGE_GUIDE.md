# How to Use This Prompt Package

## Overview

חבילת פרומטים זו מאפשרת לך לבנות את מערכת ה-NOC Agent בצורה מובנית ועקבית.
במקום לנסות לבנות את הכל בסשן אחד (שייכשל), כל שלב בנוי כסשן ייעודי עם 
מטרה ברורה.

## Prerequisites

1. **העתק את `00_master/CLAUDE.md` לשורש הפרויקט שלך.** Claude Code קורא אותו 
   אוטומטית. אם אתה עובד ב-claude.ai, הצמד אותו כcontext לכל סשן.

2. **קרא את ה-PDF המלווה** (`NOC_Agent_Architecture_Plan.pdf`) כדי להבין את 
   התמונה הכוללת.

3. **הכן את הסביבה:**
   - PostgreSQL 15+ עם pgvector extension
   - Redis
   - Python 3.11+
   - Node.js 20+ (לfrontend)
   - גישה ל-Anthropic API (key)
   - גישה ל-Jira עם admin permissions (ליצירת custom fields)

## Workflow מומלץ

### לכל שלב:

1. **פתח סשן חדש** ב-Claude Code / claude.ai
2. **וודא ש-CLAUDE.md זמין** (הצמד אם לא אוטומטי)
3. **הדבק את תוכן `PROMPT.md`** של השלב
4. **תן לClaude את הקוד הקיים הרלוונטי** (אם יש)
5. **עבוד בפעולות קטנות** - בקש סקירה לפני כל implementation גדול
6. **בדוק את ה-CHECKLIST** בסוף השלב
7. **commit לגיט** לפני המעבר לשלב הבא

### סדר עבודה מומלץ:

```
01 → 02 → 03 → 06 → 04 → 05 → 07  (backend)
                         ↓
                         08 → 09 → 10 → 11 → 12  (frontend)
                                              ↓
                                              13  (autonomy, רק אחרי 2-3 חודשים)
```

**למה 06 (Jira) לפני 04 (IKB)?** כדי שמהיום הראשון הצוות ימלא שדות feedback 
ב-Jira. זה מזין את ה-IKB שיבנה בשלב 04.

## טיפים לעבודה מוצלחת

### טיפ 1: שמור את ההקשר

בתחילת כל סשן חדש, הזכר לClaude:
```
אני ממשיך פיתוח של NOC Agent. ראה CLAUDE.md בשורש הפרויקט לקונטקסט. 
השלבים 01-03 הושלמו. היום אני עובד על שלב 04 - IKB.
```

### טיפ 2: שתף קוד קיים כשרלוונטי

במיוחד בשלב 02 (Tool Registry), שתף את הקוד שלך:
```
הנה pcap_session_checker.py הקיים שלי. איך לעטוף אותו ב-Tool abstraction 
החדשה?
[paste code]
```

### טיפ 3: בקש סקירה לפני implementation

במקום "תכתוב לי את כל X", עדיף:
```
לפני שאתה כותב קוד, תן לי את התכנון ברמה גבוהה - מבנה קבצים, public 
interfaces, ו-data flow. אחרי שנאשר את התכנון, נעבור לimplementation.
```

### טיפ 4: עבוד מודולרית

גם בתוך שלב, אל תבקש הכל בבת אחת:
```
נתחיל עם הTool abstraction (base.py + registry.py) בלבד. אחרי שזה 
עובד, נעבור ל-executor ו-3 הדוגמאות.
```

### טיפ 5: הפעל tests מוקדם

בכל שלב שיש CHECKLIST - עבור עליו בפועל, לא רק על הנייר. אם tests נופלים, 
תקן לפני מעבר לשלב הבא.

## טיפול בבעיות נפוצות

### "Claude שוכח את ההקשר"

- וודא ש-CLAUDE.md בכל סשן
- קצר פרומטים גדולים במידת הצורך
- חלק משימות לסשנים נפרדים

### "הקוד לא מתאים לסטנדרטים"

- הזכר במפורש: "כל הקוד חייב לעמוד בסטנדרטים מ-CLAUDE.md: type hints 
  מלאים, async/await, structlog, explicit exceptions"

### "השלב יותר מדי גדול"

- חלק ל-sub-goals. לדוגמה שלב 05 אפשר לפצל ל:
  * 5a: Checklists + prompts
  * 5b: Tool adapter + Anthropic integration
  * 5c: Investigation loop + parser
  * 5d: Persistence + tests

### "אני לא בטוח איך להתחיל שלב"

- תקרא את השלב הקודם ב-CHECKLIST - וודא שהכל עובד
- קרא את ה-PROMPT.md של השלב כולו, אל תתחיל לפני
- תשאל את Claude: "בהתבסס על PROMPT.md של שלב X, מה צריך להיות המחשבה 
  הראשונה שלי? מאיפה להתחיל?"

## מתי לסטות מה-plan

הפרומטים האלה הם המלצה מבוססת best practice, לא תורה מסיני. סטה אם:

1. **יש לך צורך ייחודי שלא שקלתי** - לדוגמה integration ספציפי לארגון
2. **גילית שגיאה בתכנון** - דון עם Claude ועדכן את הפרומט
3. **הקוד הקיים שלך דורש גישה שונה** - התאם

עדכן את CLAUDE.md בהתאם כשמקבלים החלטות שונות.

## שאלות נפוצות

**Q: האם אני חייב לעשות את כל השלבים?**
A: לא. 01-07 + 08-10 נותנים לך מערכת מלאה ושמישה. 11-13 הם שדרוגים.

**Q: כמה זמן זה לוקח?**
A: לפי ההערכה ב-PDF: MVP (01-07) ~2-2.5 חודשים. מערכת לומדת (01-10) ~4-5 
חודשים. Autonomy (01-13) ~6-7 חודשים + 2-3 חודשי data collection.

**Q: אני לבד בפיתוח?**
A: כן - הפרומטים מותאמים לcontributor יחיד. אם יש צוות, אפשר להריץ שלבים 
במקביל (למשל 06 ו-04 בו זמנית).

**Q: מה עם auth / security?**
A: בכוונה לא הכנסתי לפרומטים - כל ארגון מגדיר אחרת. אחרי שלב 08 (UI), 
הוסף middleware auth לפי הסטנדרטים של הארגון שלך.

**Q: האם אני יכול להשתמש ב-Claude אחר (Opus / Sonnet)?**
A: כן. המלצה: Sonnet 4.5+ לעבודה יומיומית, Opus לאתגרי תכנון/דיבאג מורכבים.
